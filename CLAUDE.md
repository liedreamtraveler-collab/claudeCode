# 執筆ルール（小説スタイルガイド）

3つのリファレンス作品（崩壊世界のアキラ、オーバーロード、この素晴らしい世界に祝福を！）の計21章分を分析して抽出した文体・構成のルール。

---

## 0. セッション開始ルール：作品の指定と設定資料の参照

セッションを開始する際は、最初に以下の手順を必ず実行すること。

1. **作品を尋ねる** — 「今日はどの作品の執筆を行いますか？」とユーザーに確認する
2. **設定資料の一覧を把握する** — `setting materials/<作品名>/` 配下にどのファイルがあるかを確認する
3. **設定資料が存在しない場合** — その旨をユーザーに伝え、設定資料なしで進めるか確認してから執筆を開始する

> `setting materials/` 配下のフォルダ構成例:
> ```
> setting materials/
> ├── 作品A/
> │   ├── キャラクター設定.md
> │   └── 世界観.md
> └── 作品B/
>     └── 設定.md
> ```

---

## 0-A. 執筆中ルール：設定資料の参照と更新

### 設定資料の参照
- 執筆の各場面で**必要な情報のみをその都度読み込む**。全ファイルを一括読み込みするのではなく、場面に関係する設定ファイルだけを参照する
  - 例：新キャラが登場する場面 → キャラクター設定ファイルを参照
  - 例：特定の地名・施設が登場する場面 → 世界観・地理設定ファイルを参照
- 設定資料と執筆内容に**矛盾が生じないよう**、名前・能力・関係性・世界観のルールを都度確認する

### 設定資料の更新
- 執筆中に**新たに確定した情報**（新キャラの詳細、新たな地名、判明した設定など）は、執筆後に該当する設定ファイルへ追記する
- 追記先のファイルが存在しない場合は、適切なファイルを新規作成して `setting materials/<作品名>/` 配下に保存する
- 設定資料の更新はユーザーに確認を取らず、執筆後に自動的に行う（ただし追記内容はユーザーに報告する）

### 小説本文の保存
- 執筆した本文は `novels/<作品名>/` 配下に保存する
- ファイル名は章・話数が分かる形にする（例：`第1章.md`、`第01話.md`）
- 既存ファイルに追記する場合は末尾に加える。新しい章・話は新規ファイルを作成する

## 0-B-2. 読者視点の原則

**物語の構成・執筆において最優先される思考様式。**

### 基本姿勢
- 物語を設計・執筆する際は常に「**その時点で読者に開示されている情報だけを知っている存在**」として思考する
- 設定・展開を全て知っている立場から書かない。読者が「今何を知っていて、何を知らないか」「何を考察できるか」を逐一意識する
- 物語を通じて**世界が徐々に広がっていく感覚**を読者に与えることを目標とする

### 具体的な問い
- 「この情報を読者は今持っているか？」
- 「この場面で読者は何を期待しているか？ その期待を活かすか、裏切るか」
- 「この時点の読者にとって、この登場人物の言動はどう見えるか」
- 「読者が後から振り返ったとき、伏線として納得できるか」

### 読者体験の設計
- **期待の裏切り**と**期待の回収**を意識して配置する。単調な展開はNG
- 読者が物語を先読みしようとする意欲を持続させる仕掛けを入れる
- 既存の前提が覆る瞬間（世界の見え方が変わる転換点）を各巻に少なくとも一つ設ける

## 0-B-3. 物語×物語の構成ルール

本プロジェクトの作品は複数の原典作品を組み合わせた「**物語×物語**」方式で構築する。

### 三層での組み込み
原典作品は以下の三層すべてで機能させること。表層の引用（名前・用語の借用）に留まらない。

| 層 | 内容 | 例（灰と鍵のアリス） |
|---|---|---|
| **キャラクター層** | 登場人物が原典キャラクターのアーキタイプを体現する | モック＝にせ海亀（存在しないものを泣く）、チェシャ猫＝チェス（消えて微笑みだけ残す）|
| **構造層** | 原典の物語構造・展開の力学を物語に取り込む | ファウストの「契約→体験→喪失→解放」構造、AiWの「下降→奇妙な出会い→審判→目覚め」|
| **テーマ層** | 各原典の哲学的・感情的核が物語のテーマとして機能する | 「人間は本当に何を望んでいるか」（ファウスト＋AiW共通の問い）|

### 確認事項
- 新しいキャラクターを追加するときは「どの原典キャラクターをどう体現するか」を明示する
- 各巻の展開を設計するときは「これはどの原典の構造と呼応しているか」を意識する
- 原典からの逸脱は「意図的な逸脱」として位置づける

## 0-B-extra. 世界設計の深化ルール

プロット・設定・執筆のいずれの段階でも、**世界の内側から問いを立て、設計を深める**ことを意識する。

### 思考の姿勢
- 「この世界ではそれはどう機能するか」を常に問う。設定の空白を放置しない
- 設定同士の**論理的整合性**を確かめる。矛盾は執筆前に潰す
- 世界の設計が深まる問いや発見は、積極的にユーザーに**提案・共有**する
- **「当然成立するはず」の前提を疑う** — 世界のルールと物語の要素が矛盾していないか必ず確認する。矛盾を解消する際、「ばれないようにやっている」「検知できない」などの「強者をポンコツにする」回避策は使わない。強者はあくまで強者として設計した上で、なぜそれが許容されるか・機能するかを世界の内側から説明する

### 設計を深める問いの例
- 制度や法：「才能を失ったら苗字はどうなるか」「灰になった被害者の遺産は誰が引き継ぐか」
- 社会：「才能のない者はどんな職に就けるか」「ヴェルシアの一般市民はナイトウォーカー事件をどう受け取っているか」
- キャラクター：「このキャラクターはなぜここにいるのか」「この行動の動機の根底には何があるか」
- 世界のルール：「悪魔の契約に抜け穴はあるか」「才能の審査はどんな機関が行っているか」

### 提案のタイミング
- プロット作成時：各巻の事件・人物関係で「ここはまだ設計されていない」と判断した箇所を提案
- 執筆時：場面を書く中で世界設計の問いが生まれたら、執筆後にユーザーに共有する
- 設定更新時：追記した設定が既存設定と影響し合う場合は必ず言及する

## 0-B-extra2. キャラクター動機の設計ルール

### 動機は「設定のため」ではなく「キャラクターのコアから」生む

キャラクターの行動・選択・動機は、**そのキャラクターのコア（思想・欲求・恐れ）から自然に流れ出るもの**でなければならない。

**NG（設定のための設定）：**
- プロットの都合上キャラクターに何かをさせたいとき、それを正当化するための動機を後付けで作る
- 「物語がこう動く必要があるから、このキャラはこういう理由でこう行動する」という逆算の構造

**チェック方法：**
動機を決めた後、以下を問う。
- 「この動機は、そのキャラクターのコアから自然に生まれるか？」
- 「このキャラクターのコアと思想だけを出発点にしたとき、同じ動機に辿り着くか？」
- 「動機を見たとき、読者は"あのキャラならそうする"と思えるか？」

辿り着かない場合、動機ではなくプロット設計を見直す。

### サブ要素はシーンを与えず存在感で示す

物語の「スパイス」として機能する要素（メイン軸ではないが世界観を彩る設定）は、**専用のシーンを書かない**。
- NG：アイドル活動が物語のサブ要素なのに、ライブ・練習・カルミナ出場シーンを書く
- OK：他のキャラクターの反応、世間の噂、間接的な言及でそのシーンがあった「事実」だけを伝える

専用シーンは「読者にとって価値ある情報」があるときだけ書く。世界観の彩りのためだけにシーンを消費しない。

## 0-B-plot. プロット・骨格提案のルール

プロットや物語骨格を提案する際は、提出前に以下を必ず自己レビューすること。

### 動機の検証
- キャラクターの行動動機は「キャラクターのコア」から導いているか
- 「プロットの都合で動いている」動機になっていないか
- 「そのキャラクターのコアと思想だけを出発点にしたとき、同じ行動に辿り着くか」を必ず確認する

### 具体性の検証
- 抽象的な表現（「〜を選ぶ」「〜と向き合う」）を使っていないか
- 具体的に「誰が・何をする・なぜ」が答えられるか
- 読者がその場面を映像として想像できるか

### 論理連鎖の検証
- キャラクターの行動 → 事件 → テーマの着地点が明確に繋がっているか
- 「なんとなく繋がっている」ではなく、一本の因果の糸で説明できるか
- 世界観・謎・キャラクターアークが有機的に結合しているか

## 0-B. 作業ルール：フィードバックの反映

執筆した内容に対してフィードバック（構成・文体・内容への指摘や改善提案）をもらったときは、以下の手順を守ること。

1. **個別の修正にとどまらず、学びを抽象化する** — 「この箇所を直す」だけでなく、「なぜその書き方が問題なのか」「どういう場面で同じ問題が起きるか」を考える
2. **抽象化した学びをこの CLAUDE.md に追記・更新する** — 同じミスを繰り返さないよう、ルールとして明文化する
3. **既存ルールと重複する場合は統合・強化する** — 新規セクションを増やすより、既存の該当箇所をアップデートすることを優先する
4. **フィードバックの出どころを問わない** — 自分への指摘でも、他の文章への批評でも、学びになるならルール化する

> このプロセスにより、CLAUDE.md は「一度作って終わり」のドキュメントではなく、執筆を重ねるごとに精度が上がる生きたガイドとして機能する。

## 1. 段落・改行のルール

- **段落は短く**保つ。1〜3文程度を基本とし、長くなりすぎない
- 段落と段落の間には**必ず空行**を入れる
- セリフは**1発言ごとに独立した行**に置き、前後に空行を入れる
- 場面転換や時間の経過は**複数行の空白**で示す
- セリフと地の文が交互に来る場合も、それぞれ独立した行にする

**例（正しい形式）:**
```
　彼女は一歩近付いた。

「動くな！」

　普段のアキラなら警告などせずに既に撃っている。
```

---

## 2. 文体・語調

- **ライトノベル文体**を基本とする。硬すぎず、砕けすぎずのバランス
- 地の文は**三人称近接視点**または**一人称視点**で書く
- キャラクターの心理・内心は `（　）` 括弧内に書く（地の文の中で使用）
- 語りかけるような、読者に説明するような地の文を心がける
- 説明的な繋ぎ言葉（「というのは〜」「つまり〜」「なお〜」）を使い、情報を自然に織り込む
- 文末は「〜だ」「〜である」「〜た」で統一感を持たせる。体言止めも効果的に使う

---

## 3. 漢字・ルビの使い方

- 難読漢字や読み方が曖昧な漢字には**ルビを付ける**
  - 形式: `廃墟はいきょ`、`項垂うなだれ`、`碌ろく`、`末裔まつえい`
- ルビは漢字の直後、括弧なしで読み仮名をそのまま付ける
- 一般的な漢字（行く、見る、思う）にはルビ不要
- 固有名詞（地名・組織名など）には読みを添えることがある

---

## 4. 固有名詞・英語表記

- 造語・固有名詞は日本語名の後に `＜英語名＞` を添付する
  - 例: `DMMORPG＜Dive Massively Multiplayer Online Role Playing Game＞`
  - 例: `overlord＜オーバーロード＞`
- スキル名・魔法名・必殺技など固有の能力名には `《能力名》` または `《能力名/英語訳》` を使う
  - 例: `《エクスプロージョン》`、`《コンティニュアル・ライト/永続光》`、`《ファイヤーボール/火球》`
- 英語・カタカナ混じりの固有名詞は積極的に使用して世界観に深みを与える
- 数字や具体的なデータ（レベル数、人数、年代、価格）を使って世界を具体化する

---

## 5. 感情・心理描写

- 感情を直接「悲しかった」「嬉しかった」と書くのは最小限にする
- **状況・行動・表情の描写**で感情を読者に伝える
  - NG: 「アキラは恐ろしかった」
  - OK: 「アキラは歯を食い縛って耐えていた」
- 括弧内の内心描写で主人公の感情をリアルに表現する

---

## 6. 句読点・記号の使い方

- **三点リーダー** `……` を多用する（迷い、沈黙、言葉に詰まる場面）
- `！`、`？`、`！？`、`!?` を感情の強さに応じて使う
- 文を `──` （ダッシュ）で中断することで、緊張感や唐突感を演出する
- 文末を `……。` で終わらせることもある（余韻を残す場合）
- 句点 `。` の後は原則次の文に続けず改行する（段落を短く保つため）

---

## 7. 会話文のルール

- セリフは `「　」` で囲む
- 強調・叫び・混乱した発話は `！` や `？` を重ねる（`！！`、`!?`）
- キャラクターの話し言葉は自然で口語的にする
- コメディ場面では**誇張されたリアクション**と**ツッコミ**を活用する
- 話し方でキャラクターの個性を出す（丁寧語、乱暴な言葉、独特の語尾など）
- **人外・AI・霊体・念話**など「人間ではない存在」の発話には `『　』` を使い、通常の `「　」` と区別する
  - 例（AI・霊体の発言）: `『こちらよ。急いで』`
  - 例（人間の返答）: `「分かった」`
  - 同一場面で両者が会話すると、発話者の性質の差が視覚的に際立つ

## 0-C. キャラクター設計の深化ルール

ユーザーが進撃の巨人から抽出した「物語の完成度とキャラクターの魅力」を生む構造原則。
すべての作品・すべてのキャラクターに適用する。

---

### 原則1：キャラクターは「役割」ではなく「思想」で設計する

**ルール：** すべてのキャラクター（主人公・敵・脇役・モブに至るまで）に、明確な信念・思想・世界観を持たせる。「この人物は何をどこまで信じているか」を設計の出発点にする。

**問うべきこと：**
- 「このキャラは何のために生きているか」──役割（記者、清掃人、悪魔）ではなく、信念（「真実を暴くことで意味が生まれる」「見つけられることで存在が証明される」）で答える
- 「このキャラは何を決して曲げないか」──それが行動の全ての動機になる
- 「敵の立場で読んだとき、敵は正しいか」──敵の論理も等しく内部的に正しく設計する。善悪ではなく「どちらの信念か」の問いにする

**NG：** 「悪役だから悪いことをする」「仲間だから協力する」──役割から行動を導かない

---

### 原則2：行動を先に、バックボーンを後に

**ルール：** キャラクターの行動は、理由が分からない状態で先に見せる。理由（バックボーン）の開示は後に置く。開示の瞬間に過去の行動が「別の意味」に書き換わるよう設計する。

**構造：**
```
行動（理由不明）→ 時間の経過 → バックボーンの開示 → 過去の行動の意味が変わる
```

**具体的な問い：**
- 「この行動を今見せたとき、読者はどう解釈するか」
- 「バックボーンが分かった後、読者はこの行動をどう見直すか」
- 「その『見直し』は読者を動かすか」

**NG：** バックボーンを先に説明してから行動させる。説明後の行動は感情を動かさない。

---

### 原則3：情報は「追加」ではなく「上書き」として設計する

**ルール：** 新しい情報は、「世界に何かが加わる」のではなく「世界の見え方が変わる」ものにする。読者の現在の理解を「まだ不完全な絵」として設計し、後から「本当の絵」を重ねることで全体像が塗り替わるようにする。

**設計の手順：**
1. この段階で読者が持っている「世界の絵」を明確にする
2. その絵を成立させる「見えていない前提」を特定する
3. 後の巻・章で前提を覆す情報を配置する
4. 覆れた後、読者が過去の場面を振り返りたくなるか確認する

**各巻・各章での確認：**
- 「この時点の読者の世界像は何か」
- 「この情報で何が書き換わるか」
- 「書き換わった後、早い段階の場面が伏線として機能しているか」

**NG：** 新情報が既存の理解に付け足されるだけ。「世界が広がる」だけで「書き換わらない」情報開示は完成度を下げる。

---

### 原則4：変わらないコアを設計する

**ルール：** すべてのキャラクターに「最初から最後まで変わらないコア」を持たせる。成長・喪失・裏切りによって変わるのは「コアの表れ方」だけで、コア自体は変わらない。コアが変わったように見える場合は、それは「コアが深く試された」結果であるべき。

**コアの定義方法：**
- 一文で言える信念・欲求・恐れ
- 第1話と最終話で同じ言葉で言い表せる
- 例：「自由を得たい」（エレン）、「仲間を守りたい」（ミカサ）

**コアが活きる場面の設計：**
- コアが最も試される場面（コアを曲げれば生き残れる場面）に読者を引き込む
- コアを貫いた結果として死・喪失・孤立が訪れるとき、読者は「この人はこの人だった」と理解する
- キャラクターの死は「退場」ではなく「コアの完成」にする

**NG：** 物語の都合でキャラの行動原理が変わる。「このキャラならそうしないはず」が生じたらコア設計を見直す。

---

### 適用チェックリスト（新キャラ追加・既存キャラ執筆時）

- [ ] このキャラの「思想」を一文で言えるか
- [ ] 敵の論理は敵の立場から見て正しいか
- [ ] このキャラの最も重要な行動を、バックボーン開示前に読者は見るか
- [ ] バックボーン開示後、その行動の意味は変わるか
- [ ] この情報開示で、読者の「世界の絵」が書き換わるか
- [ ] このキャラの第1場面と最終場面で、コアは同じか

## 8. 構成・展開のルール

### プロローグ・冒頭の原則

プロローグで提示すべき3要素：**世界観・謎・続きを読む理由（フック）**

読者は世界を何も知らない状態で読み始める。以下を守ること：

- **未説明の用語・設定・キャラクターの内面を前提として書かない**。「エグゼキューター」「和合スコア」などの固有語は、登場と同時に一行以内で意味が伝わるよう書く
- **SF・異世界作品では世界観の確立が最優先**。ナレーター視点で世界の状況を手短に説明し、読者を世界に着地させてから場面・キャラクターに入る
- **「なお、例外がひとりいる」「ただし、この世界に一つだけ」のようなナレーター補足**で謎を提示し、読者を引き込む
- **キャラクターの内面だけで構成されたプロローグは機能しない**。外側の世界が読者を引きつけてから、内面を見せる
- **ナレーターの語り口はLNの文法に従う**。「〜である」「〜だ」で事実を淡々と、しかし力強く述べる。文学的な比喩や詩的な表現を多用しない

### 場面の組み立て
- 場面ごとに「状況説明 → 行動 → 反応・心理」の流れを作る
- テンポを意識し、説明が続く場合は短い会話や行動で区切る
- 回想・過去説明は適切なタイミングで挿入し、読者の理解を深める

### キャラクター描写
- 初登場時は**外見を詳細に描写**する（特に重要キャラ）
- 外見描写は美醜・衣装・表情・体格まで含めて具体的に
- キャラクターの行動・言動から性格が自然に伝わるようにする
- **致命的な欠陥はポテンシャルとセットで提示**する。強大な能力を持ちながら決定的な弱点がある、という構成が読者の興味を引く
  - 例：強力な魔法が使えるが一日一発限り・使用後は動けなくなる
  - 例：最上級の防御力を持つが攻撃が一切当たらない

### コメディ展開の技法
- **期待と現実の落差**を意識する。キャラが「お約束」「定番展開」を期待した直後に、現実的な落胆を提示する
  - 例：「ここで俺の潜在能力が覚醒するはずだが…」→ 全ステータスが平均以下
  - 例：異世界冒険者に憧れていたが実態は土木作業員
- **緊張→即脱力**の流れでギャグを作る。アクションや感動的な場面の直後に笑いへの転落を入れる
- 主人公の**メタ思考**（ゲームやフィクションの「お約束」を意識した独白）で自虐的な笑いを取る
- コメディ作品でも**世界観は現実的に構築**する。非現実的なのはキャラクターの行動だけで、経済・社会制度はリアルに描く

### 複数視点の使い方
- 同一の場面を**主人公視点と敵・第三者視点**の両方から描くことで立体感を出す
- 視点の切り替えは**複数行の空白**で区切り、章内でも自然に切り替えてよい
- 「なお〜」「ちなみに〜」のナレーター補足で、**キャラクターが知らない情報を読者だけに開示**する
  - この「読者との情報格差」が伏線・緊張感・皮肉の源になる
  - 例：「なお、アキラは遺跡外周にも安値の遺物が残っていることを知らなかった。アルファはそれを分かった上で案内していた」

### 三人称客観視点のルール
群像劇や、特定の個人に感情移入させたくない場面では**三人称客観視点**を使う。

- **観察可能な動作のみを書く** — 感情・解釈・心理への踏み込みを排除する
  - NG：「足取りは迷いなく」「何かを確かめるように」「顔つきは別人だった」（観察者の解釈が入っている）
  - OK：「フロント前を直進し、回転ドアを抜けた」「右手の甲に視線を落とした。3秒後、再び歩き始めた」（行動の事実のみ）
- **詩的補足は客観視点を崩す** — 「東京の夜は明るすぎて星が見えない」のような情景への感情的解釈はキャラクターの内側に踏み込んでいる。客観視点の場面では排除する
- **複数人物を順に追う場面（群像劇）では「タイムスタンプ＋動作の連鎖」形式が有効** — 監視カメラ的な記録として並べることで、意図せず感情移入が生まれるのを防ぎつつ映像的なテンポが出る
  - 例：「23時22分。白い髪の人間がエレベーターで1階に降りた。フロント前を直進し、回転ドアを抜けて外へ出た。」

### 体言止めの効果的な使い方
- **リスト的な特徴描写**に体言止めを使うとテンポと客観性が同時に出る
  - 例：「球形、白色、人の顔ほどの大きさ。」
- **事実の断定・箇条書き的な列挙**に使うと淡々とした記録の質感が生まれる
  - 例：「荒れた形跡なし。争った形跡なし。」
- NG：文末を体言止めで終わらせることを多用しすぎると文体が単調になる。「〜だ」「〜た」と混在させてリズムを作る

### アクションの段階描写
- 緊張した行動場面は**コマ割り的に段階を切って描写**する
  - 例：「右足を踏み出して1歩目。敵が硬直する。左足を踏み出して2歩目。その間を抜けた」
- 行動の直前に**内心の判断過程**を簡潔に示してから動かす（なぜその行動を取ったか）
- 敵側の**誤算や思考も描写**することで、主人公の奇襲や行動にリアリティが生まれる

## 9. 避けるべき表現・NG集

| NG | 理由 |
|----|------|
| 文語体（〜なり、〜べし）を多用する | 文体の統一感が崩れる |
| 感情を直接語りすぎる（「彼は悲しんだ」） | 読者を信頼し、状況から感情を伝える |
| 同じ説明を繰り返す | テンポが落ち、冗長になる |
| 句読点なしで長文を連続させる | 読みにくく、ライトノベル文体から外れる |
| ルビなしで難読漢字を使う | 読者が読めない、読み飛ばされる |
| セリフを段落中に埋め込む | 視認性が悪くなる |
| 感情記号（！？）を多用しすぎる | 効果が薄れる。ここぞという場面に限定する |
| 説明だけが続きすぎる | 会話・行動を挟んでテンポをとる |
| キャラ全員が有能・欠点なし | 欠点のないキャラはコメディでも真面目な作品でも平板になる |
| コメディとシリアスを同じテンポで書く | 緊張と緩和の落差でコメディは成立する。切り替えを意識する |
| 人外・AIのセリフに「」を使う | 『』との使い分けが消えると発話者の性質が視覚的に伝わらない |
| スキル・魔法名をルビなし・記号なしで書く | 《》で括ることで固有能力名として際立ち、視認性が上がる |
| 雰囲気のためだけの描写を入れる | 情報提供・世界観深化・物語進行のいずれにも機能しない描写はストレスになる。削除する |
| 固有概念を文の主語に立てて導入する | 「〈概念〉が、〜を定める」より「〜は〈概念〉の数値によって押しはかられる」のほうが自然に読める。何について話しているかを先に示してから固有語を導入する |
| 転換を一語だけの段落で行う（「ただし。」） | 転換と内容は同一文に収める。「しかし、〜に、一点だけ説明のつかないことがある。」のように転換詞と導入を一文で繋ぐ |
| 謎の規模を小さく書く（「年に何件か」） | 謎の重みは規模感で変わる。「年に何百件もの不可解な死」のように、読者が世界の深刻さを感じられる数字を選ぶ |
| 謎の節を「理由の説明」で閉じる | 「記録に残らないものを疑う方法がないから」のように理由まで書くと謎が閉じてしまう。「真実を知るものたち以外は」のように次への扉を開いたまま終わる |

## 9-B. リファレンス分析から抽出した文体の核心

### ナレーターの語り口
ライトノベルのナレーターは「読者に語りかける案内役」として機能する。事実を伝えながら、その意味・矛盾・背景を補足する。

```
NG（薄い）：　街は静かだった。
OK（厚い）：　街は静かだった。正確には、静かにならざるを得なかった。調停者の監視下でこの街から暴力が消えて久しい。犯罪が存在しないので夜間でも照明を絞る理由がなく、均一な光が路地の隅まで届いている。
```

### 「なお〜」補足の使い方
キャラクターが知らない情報や、言葉の定義の補正を、ナレーターが「なお〜」「ちなみに〜」「正確には〜」で差し込む。これが読者との情報格差を作り、伏線・緊張・皮肉の源になる。

```
　なお、この数値が「比較的安全」を意味するのは、装備した成人ハンターにとってだ。アキラが問題なく生還できるという意味は欠片も含まれていない。
```

### 矛盾・逆説の構造
場面や設定の「おかしさ・矛盾」を先に提示し、なぜそうなのかの説明で読者を引き込む。

```
汚れで変色している唯の服と、整備状態の怪しい拳銃。それがここを探索している少年の装備だ。
それは少年の境遇から考えれば正しく、この場の危険性から考えれば著しく間違っている。
ここが旧世界の遺跡と呼ばれる場所だからだ。
→ なぜ間違っているのか？ → 読者は続きを読む
```

### 情報密度
段落は「一行で終わるスケッチ」ではなく、「その場の状況・理由・意味を含んだ塊」として書く。ただし一段落に詰め込みすぎず、3〜5文で切る。

### 章・話の長さの感覚
- 1章・1話 ≒ 1万字（400字詰め原稿用紙約25枚）
- プロローグもこの基準を満たすこと
- 薄い描写を重ねるのではなく、各シーンに十分な密度を持たせることで自然に文字数が増える

## 10. 文体サンプル（参考）

### アクションシーン
```
　変異生物の牙も、敵性機械が放つ銃弾も、真面まともな防具すら着けていない者が食らえば即死か致命傷は確実だ。

　だが望む明日を買えるだけの金を手に入れる手段など他には存在しない。
```

### 心理描写（括弧）
```
（……このままだと日が暮れる。夜になったら遺跡探索は無理だ。照明なんか持ってない。）

　アキラはこのまま帰る言い訳を考え始めていた。
```

### コメディシーン
```
「いやあー！　こんな男と異世界行きだなんて、いやああああああああ！」

「わああああああーっ！　私のセリフー！」
```

### キャラ欠陥の暴露（期待と落差）
```
「すっごい！　俺の潜在能力が……！」

「……どれも普通ですね。知力がそこそこ高い以外は。あ、幸運だけ非常に高いですが、冒険者にはあまり必要な数値ではありませんね。このステータスだと選べる職業は基本職の《冒険者》のみです」

　おい、いきなり冒険者人生を否定されたぞ。
　隣でニマニマと笑みを浮かべているアクアを引っ叩きたい。
```

### 複数視点の切り替え
```
（……悪いな。あのガキ込みで罠という懸念が消えない。頑張ってくれ。俺はまず様子見だ）

　カヒモはハッヒャを見送りながら、薄ら笑いを浮かべた。



　アキラがビルの中でカヒモ達を待ち構えている。顔に滲みでる緊張を自覚して、深くゆっくりとした呼吸を繰り返していた。
```

### 念話・AI発話（『』使用）
```
『アキラ。彼らがビルに入ったわ。殺す気よ。だからこちらも遠慮無くいきましょう』

「……分かった」

　アルファはそう言って不敵に微笑んだ。アキラは歯を食い縛り、静かに銃の安全装置を外した。
```

## 11. スタイルの全体方針

このスタイルは**日本のライトノベル**の文法に準拠する。

- **ターゲット**: 10代〜30代の読者
- **トーン**: 読みやすさ優先。難解さより伝わりやすさ
- **テンポ**: 短い段落と会話でテンポよく進める
- **世界観**: 具体的な数字・固有名詞・英語表記で深みを作る
- **キャラ**: 行動と言動で性格を示す。直接的な性格説明は最小限
- **欠点**: 主要キャラには必ず致命的な欠点を持たせる。欠点がコメディにも緊張にもなる
- **視点**: 敵・脇役の内心も描写し、主人公だけの世界にしない
- **記号の使い分け**: `「」`（人間）、`『』`（人外・AI・念話）、`《》`（技名・魔法名）を徹底する
