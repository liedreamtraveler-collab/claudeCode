# Novel Design Workflow

Design story elements through structured questions before proposing anything.

## Core Principle
Never propose complete solutions without first raising resolution on what the user finds interesting. The quality of design depends entirely on the quality of questions asked.

## Workflow

Make a todo list for all tasks in this workflow and work through them one by one.

### 1. Load Context
- Check what files exist in `setting materials/<work-name>/`
- Identify what is already defined vs. what is blank
- If no setting files exist, ask for the work name and create the directory

### 2. Identify the Design Task
Confirm with the user: what specifically are we designing today?
- Plot / story structure
- Character arc or motivation
- World-building element
- Theme or concept
- Scene or chapter breakdown

### 3. Question Session (most critical step)
Ask 2–3 focused questions before proposing anything. Good questions:
- Raise resolution on what the user finds "interesting" or "wants"
- Probe intuition even when the user hasn't formed a clear preference
- Uncover hidden constraints or desires

Do NOT propose a complete design until the user's answers give clear direction.

### 4. Propose Structure
Based on answers, propose concrete structure using the CLAUDE.md frameworks. Before finalizing, verify:
- Is each character's action driven by their core, not plot convenience?
- Does new information rewrite the reader's prior understanding (not just add to it)?
- Is there a clear causal chain connecting action → event → theme?
- Does the world design have internal logical consistency?

### 5. Update Setting Files
After design is confirmed, update or create files in `setting materials/<work-name>/`.
Report all additions and changes to the user.

## Wrap Up
Summarize: what was designed, what files were updated, and what open questions remain for the next session.
