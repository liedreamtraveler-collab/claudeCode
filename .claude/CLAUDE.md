# Global Rules

## Language Policy
- All rule and instruction files (CLAUDE.md, SKILL.md): write in English
- Creative content (novel text, lyrics): stays in its original language (Japanese)
- Code comments: English

## Domain Map

| Domain | Path | Status |
|--------|------|--------|
| Creative (novels, lyrics) | /home/user/claudeCode/ | Active |
| Code (dev, design) | ~/projects/code/ | Create when needed |
| Content (X posts, research) | ~/projects/content/ | Create when needed |

## Cross-Domain Principles

1. **Question first** — ask clarifying questions before proposing complete solutions
2. **Feedback loop** — after each session: extract learnings → abstract into rules → update the relevant domain CLAUDE.md
3. **No repeat mistakes** — never repeat an error already captured in any CLAUDE.md
4. **Domain isolation** — each domain CLAUDE.md is authoritative for its domain; global rules apply everywhere

## Growing This System

- Domain CLAUDE.md files evolve independently through the feedback loop
- Skill files evolve as workflows are refined through repeated use
- When adding a new domain, create its directory + CLAUDE.md before starting work
- When a skill consistently produces friction, update its SKILL.md
